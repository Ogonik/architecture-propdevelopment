# Задание №2

## Обоснование включения тех или иных разделов в чек-лист ИБ
**I. Управление доступом**	- Раздел актуален для чек-листа безопасности, т.к. к системам компании обращаются пользователи (Клиенты и Сотрудники)

**II. Безопасность данных**	- Раздел актуален для чек-листа безопасности, т.к. системы компании содержат компоненты перситентного хранения данных

**III. Защита инфраструктуры**	- Раздел актуален для чек-листа безопасности, т.к. компания использует собственную инфраструктуру

**IV. Управление инцидентами**	- Раздел актуален для чек-листа безопасности, т.к. исходя из анализа рисков ИБ (Задание 1), существуют критические риски для функционирования компании, поэтому ИТ-системы управления инцидентами ИБ необходимы компании

**V.1 Специфические аспекты. Общие вопросы** - Раздел актуален для чек-листа безопасности, т.к. представленные вопросы корреспондируют с выявленными рисками ИБ

**V.2 Специфические аспекты. Управление оборудованием и физической безопасностью** - Раздел актуален для чек-листа безопасности, т.к. представленные вопросы корреспондируют с выявленными рисками ИБ
